#  Testing - Day 4
Welcome to **day 99** of 365 days of code - coding every day for a year, little and often

Another mixed bag of a day today. I wanted to work out the rest of the data.ts test cases, but ended up first getting sucked into updating next.js on the main build as there was an update that remediated a CVE so I wanted to get that done.

Well, updating it, not a hard thing, in fact relatively easy, however... I then tried to merge the main back into the feature branch I'm working on for the testing and I got stuck in a world of git pain as I realised I hadn't pushed my local changes to the remote repo, before creating the pull on the remote repo, so things just got messy quickly. And as we all know when untangling a ball of wool, you just have to take it slowly and one line/knot at a time.

Anyway, that all being sorted out, it was time to sort out the test cases for the data.ts file. Honestly I am leaning on AI, in particular Claude, for both scoping the test cases, and writing them. I haven't got the vernacular down just yet, and it feels quite complex to be writing these so deep into the project, but I am picking things up along the way, my approach is to have a go at writing it myself first, try it out, and then feed it into Claude to get it to check for issues with the way I've written it, and look for gaps that I haven't covered in my testing. It feels a little like cheating at times, but at least I'm not just feeding it the whole codebase and saying "Go!"...

The end result is that the data.ts test cases are fully written now and working, so that's good, I suppose I should tackle the actions one next, although that feels a bit more daunting, so maybe I'll pick some lower hanging fruit first.

Anyway, more tomorrow!
