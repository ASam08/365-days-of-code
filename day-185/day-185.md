#  Sidetracked
Welcome to **day 185** of 365 days of code - coding every day for a year, little and often

Another day, and today was a bit more finicky, I went through and tidied up the extra code left behind int he dashboard card components/data fetches, but I had to think really carefully about security on these. I don't want there to be the possibility that someone can manipulate the component requests and get someone elses data, so like I said, a fair bit of time thinking about it. This was when using Claude as a pair programmer came back to me. If I've written the code myself, and reviewed it myself, it's easy to miss things, so getting Claude to review it and check for gaps or potential security issues is pretty helpful. I don't just give it the code and say fix it, and Claude isn't actually linked into my project, I'm just pasting code into the prompt with a request to review it for security/data leaks and it comes back to me with any issues and then I can go and fix them myself.

I know everyone is all AI this and AI that, and obviously if I'm doing 365 days of code, I want to be writing it myself, that's sort of the whole point, so this feels like a happy middle ground. While I was doing this, Claude actually spotted a security gap in the unhideDow actions, so I tidied that up too.

Anyway, I also went through and checked that there were no issues or gotchas with the edit timetable block flow but that all looks good as it actually uses the blockID as the reference and then checks that the user owns the timetable set that the block belongs to before changing it, so I'm happy there.

I also got a little sidetracked, because I was looking at the dashboard card pieces, I ended up condensing the code on these down, I had 3 actions, one for each card type, but they were pretty much doing exactly the same thing, so I combined them and that reduces some code, not a bad thing.

And then there was nothing to do but tests... more tomorrow I guess!
